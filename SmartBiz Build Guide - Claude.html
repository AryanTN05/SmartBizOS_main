M1: AI Business Assistant (RAG Chatbot)

Aarsh owns AI pipeline. Vinayak owns chat UI. Vivaan owns API endpoints.

How RAG works in this system

Document Ingestion (write path)

  • User uploads PDF/DOCX
  • PyMuPDF extracts clean text
  • Text split into 500-token chunks with 50-token overlap
  • Each chunk embedded via OpenAI text-embedding-3-small
  • Vectors stored in Pinecone with metadata
  • Document record saved to PostgreSQL

Query (read path)

  • User sends question via chat
  • Question embedded → vector search in Pinecone
  • Top 5 relevant chunks retrieved
  • Claude receives: system prompt + chunks + chat history + question
  • Response streams back via Server-Sent Events
  • Sources attributed to specific documents

import fitz # PyMuPDF from openai import AsyncOpenAI from pinecone import Pinecone from app.core.config import settings import uuid openai_client = AsyncOpenAI(api_key=settings.OPENAI_API_KEY) pc = Pinecone(api_key=settings.PINECONE_API_KEY) index = pc.Index(settings.PINECONE_INDEX_NAME) async def ingest_document(file_bytes: bytes, filename: str, doc_id: str): """Extract text, chunk, embed, store in Pinecone""" # 1. Extract text from PDF pdf = fitz.open(stream=file_bytes, filetype="pdf") full_text = "" for page in pdf: full_text += page.get_text() # 2. Chunk into 500-token pieces with 50-token overlap words = full_text.split() chunks, i = [], 0 while i < len(words): chunk = " ".join(words[i:i+500]) chunks.append(chunk) i += 450 # 50-token overlap # 3. Embed all chunks (batch for efficiency) embeddings = await openai_client.embeddings.create( input=chunks, model="text-embedding-3-small" ) # 4. Upsert to Pinecone vectors = [] vector_ids = [] for i, (chunk, emb) in enumerate(zip(chunks, embeddings.data)): vid = f"{doc_id}-chunk-{i}" vectors.append({ "id": vid, "values": emb.embedding, "metadata": { "doc_id": doc_id, "filename": filename, "text": chunk, "chunk_index": i } }) vector_ids.append(vid) index.upsert(vectors=vectors) return vector_ids async def query_rag(question: str, session_history: list, k: int = 5): """Query Pinecone, build context, stream Claude response""" import anthropic # 1. Embed the question q_emb = await openai_client.embeddings.create( input=[question], model="text-embedding-3-small" ) # 2. Search Pinecone for top-k relevant chunks results = index.query( vector=q_emb.data[0].embedding, top_k=k, include_metadata=True ) # 3. Build context string from retrieved chunks context_parts = [] sources = [] for match in results.matches: if match.score > 0.70: # Only use relevant results context_parts.append(match.metadata["text"]) sources.append({ "filename": match.metadata["filename"], "score": round(match.score, 2) }) context = "\n\n---\n\n".join(context_parts) # 4. Build messages for Claude system = f"""You are a helpful business assistant. Answer questions using ONLY the provided context. If the answer isn't in the context, say so clearly. CONTEXT: {context}""" messages = session_history + [{"role": "user", "content": question}] # 5. Stream Claude response client = anthropic.Anthropic(api_key=settings.ANTHROPIC_API_KEY) with client.messages.stream( model="claude-sonnet-4-6", max_tokens=1024, system=system, messages=messages ) as stream: for text in stream.text_stream(): yield text, sources

M3: Lead Nurture Automation

Aryan owns this entirely. Built in n8n. Triggered by M2's webhook when a new lead is created.

1
n8n workflow — New Lead → AI Email → Slack → Drip
12 hrs — Aryan
▼

n8n nodes in order

  • Node 1 — Webhook trigger: POST /webhook/new-lead. Receives: id, name, email, company, message, source.
  • Node 2 — Claude email personalisation: HTTP POST to Claude API (Haiku). Prompt: "Write a 2-sentence warm intro email opening to {{name}}, who reached out about: {{message}}. Be friendly, direct, under 30 words."
  • Node 3 — Email send (Day 0): Send via Zoho SMTP. From: hello@zerotoprod.tech. Subject: "Re: your message, {{name}}". Body: hook + standard template.
  • Node 4 — Slack #leads alert: Post formatted message with all lead details + a "View in CRM" deep link button.
  • Node 5 — Wait (3 days): n8n Wait node, 72 hours.
  • Node 6 — Check if status changed: GET /api/leads/{{id}} from FastAPI. IF status !== "new" → stop (they replied or were contacted manually).
  • Node 7 — Email send (Day 3): Follow-up email template.
  • Node 8 — Wait (4 more days): Another Wait node.
  • Node 9 — Final check + Day 7 email: Same status check → breakup email if still new.
Important: Set n8n to use your N8N_LEAD_WEBHOOK URL. Add this to the FastAPI .env and to the backend config. The lead creation endpoint POSTs to this URL — Aryan needs to give Vivaan the webhook URL once it's set up in n8n.

import fitz import anthropic import json from app.core.config import settings EXTRACTION_PROMPT = """Extract all key information from this document. Return a JSON object with these fields: - document_type: (contract/invoice/report/letter/other) - parties: [list of companies or people mentioned] - dates: {created, effective, expiry, deadline} (ISO format or null) - amounts: [list of {amount, currency, description}] - key_clauses: [list of important clauses or terms, max 5] - action_items: [list of required actions] - anomalies: [list of unusual or concerning items] - summary: one paragraph plain-language summary Return ONLY valid JSON, no markdown, no explanation. DOCUMENT TEXT: {text}""" async def extract_document(file_bytes: bytes, filename: str) -> dict: # Extract text pdf = fitz.open(stream=file_bytes, filetype="pdf") text = "" for page in pdf: text += page.get_text() # Truncate if too long (Claude context limit) text = text[:15000] # Call Claude client = anthropic.Anthropic(api_key=settings.ANTHROPIC_API_KEY) response = client.messages.create( model="claude-sonnet-4-6", max_tokens=2000, messages=[{ "role": "user", "content": EXTRACTION_PROMPT.format(text=text) }] ) # Parse JSON response try: extracted = json.loads(response.content[0].text) except: extracted = {"error": "extraction failed", "raw": response.content[0].text} return extracted
